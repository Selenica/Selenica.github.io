概述

Data Lab 的目的是让读者熟悉常见模式和整型的位级表示。这个实验将提供一系列的编程谜题，需要读者在受限条件下解决。所有谜题如下表所示：

函数 描述
copyLSB(x) 如果 $x$ 的 LSB 为 0 则返回全 0，否则返回全 1
allOddBits(x) 判断 $x$ 所有奇数位是否都是 1
isNotEqual(x, y) 判断 $x$ 和 $y$ 是否不相等
dividePower2(x, n) 计算 $x \div 2^n$
remainderPower2(x, n) 计算 $x\text{ mod } 2^n$
rotateLeft(x, n) 将 $x$ 左旋 $n$ 位
bitMask(highbit, lowbit) 生成一个 lowbit ~ highbit 的掩码
isPower2(x) 检查 x 是否是 $2$ 的幂
allAsciiDigits(x) 检查 $x$ 的所有字节是否在 '0' ~ '9' 之间
trueThreeFourths(x) 计算 $x \times 3 \div 4$，向 $0$ 取整
bitCount(x) 计算 $x$ 中 1 位的数量

实验手册和实验材料可以在这里查阅。

第一部分

最低有效位

如果 $x$ 的最低有效位为 0，则返回全 0，否则返回全 1。

C 语言对有符号数的右移操作是算数右移。只需用 x & 1 获取 LSB，再左移 63 位、右移 63 位即可。

所有奇数位

判断长整型数 $x$ 的所有奇数位是否都为 1。

首先构造奇数位掩码 0xaaaaaaaaaaaaaaaa（0b10..1010），再判断 $x$ 在掩码上的位是否都是 1。

不相等

返回逻辑表达式 x != y 的值。

使用 !!expr 得到一个整数表达式的布尔值。

计算 $x \div 2^n$

计算 $x \div 2^n$，向 $0$ 取整。其中 $0 \leqslant n \leqslant 62$。

如果 $x \geqslant 0$，则直接右移 $n$ 位。如果 $x < 0$，为确保向 $0$ 取整，先加 1..1 再右移 $n$ 位。这里 (1L << n) + ~0L 构造了一个低 $n$ 位全 1 的掩码。

第二部分

计算 $x\text{ mod } 2^n$

计算 $x\text{ mod } 2^n$，如果 $x < 0$ 应当返回负余数。其中 $0 \leqslant n \leqslant 30$。

x = (~flag & x) | (flag & (~x + 1)) 是选择赋值语句的替代：如果 $x \geqslant 0$，则直接取 $x$ 的低 $n$ 位；否则先令 $x := -x$ 再取低位，最后返回 $-x$。

左旋 $n$ 位

将长整型数 $x$ 左旋 $n$ 位。其中 $0 \leqslant n \leqslant 63$。

1L << 63 >> (n + ~0L) 形成一个高 $n$ 位全为 1 的掩码；1L << n 得到一个第 $n$ 位（从 $0$ 开始）为 1，其余位全为 0 的数，将其值减 $1$ 得到一个低 $n$ 位全为 1 的掩码。接下来需要将 $x$ 左移，高位右移补充即可。

生成掩码

生成一个从 $a$ 到 $b$ 的掩码。其中 $0 \leqslant a, b \leqslant 63$。如果 $a > b$，掩码应为全 0。

(2L << highbit) + ~0L 生成一个低 $\text{highbit} + 1$ 位全为 1 的掩码（参考上题），将其右移 $\text{lowbit}$ 位再左移 $\text{lowbit}$ 位即可得到所需的结果。

判断幂

判断 $x$ 是否是 $2$ 的整数次幂。注意负数不可以是 $2$ 的幂。

!!x & !(x >> 63) 的含义是判断 $x$ 是否为正数，!((x + ~0L) & x) 等同于 ((x - 1) & x) == 0，只有当 $x$ 为 $2$ 的整数次幂时值才为真。

第三部分

每个字节都满足条件

判断长整型数 $x$ 的每个字节 b 是否都满足 0x30 <= b && b <= 0x39。

这一题不允许使用 + 运算符，所以不能使用相减观察符号位的方法。考虑直接从 0x30 到 0x39 的位特征入手。观察可知符合条件的位的正则表达式为 /(00110...)|(0011100.)/，其中 . 是 0 或 1。

根据以上的位特征得到算法思路：首先匹配每个字节的高 4 位（[7..4] 位），如果这四位不是 0011 则匹配失败；再匹配每个字节的第 4 位，如果这一位是 0 则匹配成功，否则继续匹配 [2..1] 位，如果这两位不是 00 则匹配失败。拓展所需的掩码以使得算法对长整型数的每个字节都有效。代码如下：

计算 $x \times 3 \div 4$

计算 $x \times 3 \div 4$，向 $0$ 取整。需要避免运算过程中溢出。

根据向 $0$ 取整的计算法则，对于 $x \geqslant 0$，有
$$
\lfloor x \times \frac{3}{4} \rfloor = 3 \lfloor \frac{x}{4}\rfloor + \lfloor (x ~\text{mod}~ 4) \times \frac{3}{4} \rfloor = \text{M} + \Delta_f
$$

对于 $x < 0$，有
$$
\lceil x \times \frac{3}{4} \rceil = 3 \lfloor \frac{x}{4}\rfloor + \lceil (x ~\text{mod}~ 4) \times \frac{3}{4} \rceil = \text{M} + \Delta_c
$$

不妨考虑下面几种取整的情况：

$$
\lfloor 0 \times \frac{3}{4} \rfloor = 0,~\lfloor 1 \times \frac{3}{4} \rfloor = 0,~\lfloor 2 \times \frac{3}{4} \rfloor = 1,~\lfloor 3 \times \frac{3}{4} \rfloor = 2
$$
$$
\lceil 1 \times \frac{3}{4} \rceil = 1,~\lceil 2 \times \frac{3}{4} \rceil = 2,~\lceil 3 \times \frac{3}{4} \rceil = 3
$$

根据上式建立低两位到 $\Delta_f = \lfloor (x ~\text{mod}~ 4) \times \frac{3}{4} \rfloor$ 或 $\Delta_c = \lceil (x ~\text{mod}~ 4) \times \frac{3}{4} \rceil$ 的映射关系：

符号 余数（低两位） $\Delta$
00 $0$
$+$ 01 $0$
$+$ 10 $1$
$+$ 11 $2$
$-$ 01 $1$
$-$ 10 $2$
$-$ 11 $3$

这种映射关系可以用 $\Delta = $ (remain + (remain << 1) + ((x >> 63) & 0x03)) >> 2 来表达。

计算 $1$ 比特数

计算长整型数 $x$ 中 $1$ 比特的个数。

由于实验不允许使用控制流结构，操作符限制在 50 个以内，所以这里计算 $1$ 比特数主要运用分治-归并的思想。分治算法的含义是将大问题划分为若干个个规模较小、结构相似的子问题。这些子问题解决的方法都是类似的，解决掉子问题之后，归并子问题的结果，就得到了大问题的解。

为了理解这一算法，不妨先试着计算 8 位整数 0xe9（11101001）中 $1$ 比特的个数，算法从两位分治开始。首先考察第一次分治的其中一个区间。这个区间长度只有两位，掩码为 01。x & mask 的含义是和低位匹配，x >> 1 & mask 的含义是和高位匹配。不难看出，(x & mask) + (x >> 1 & mask) 的结果就是 $x$ 的这两位中 1 的个数。对四个区间都进行相同的操作，最终的结果是 10010101，也即 $[2, 1, 1, 1]$。第一次计算流程如下表：

07 06 05 04 03 02 01 00 HEX
x 1 1 1 0 1 0 0 1 0xe9
x >> 1 1 1 1 1 0 1 0 0 0xf4
mask 0 1 0 1 0 1 0 1 0x55
x & mask 0 1 0 0 0 0 0 1 0x41
x >> 1 & mask 0 1 0 1 0 1 0 0 0x54
+ 1 0 0 1 0 1 0 1 0x95
归并结果 2 1 1 1 5

第一次归并之后，我们已经把结果分散存放在不同的区间内。接下来对上一次归并产生的结果（10010101）继续使用四位分治，一个区间的掩码为 0011。(x & mask) + (x >> 1 & mask) 的含义就是把两个第一次归并的区间的值相加，得到每四位中 1 的个数。对两个区间都进行相同的操作，最终的结果是 00110010，也即 $[3, 2]$。第二次计算流程如下表：

07 06 05 04 03 02 01 00 HEX
x 1 0 0 1 0 1 0 1 0x95
x >> 2 1 1 1 0 0 1 0 1 0xe5
mask 0 0 1 1 0 0 1 1 0x33
x & mask 0 0 0 1 0 0 0 1 0x11
x >> 2 & mask 0 0 1 0 0 0 0 1 0x21
+ 0 0 1 1 0 0 1 0 0x32
归并结果 3 2 5

仿照上面的操作继续计算。这一次的归并区间长度为 8，是最后一次归并。得到结果 $[5] \to 5$ 即为 8 位数中 1 的个数，计算结束。第三次计算流程如下表：

07 06 05 04 03 02 01 00 HEX
x 0 0 1 1 0 0 1 0 0x32
x >> 4 0 0 0 0 0 0 1 1 0x03
mask 0 0 0 0 1 1 1 1 0x0f
x & mask 0 0 0 0 0 0 1 0 0x02
x >> 4 & mask 0 0 0 0 0 0 1 1 0x03
+ 0 0 0 0 0 1 0 1 0x05
归并结果 5 5

事实上，以上分治-归并算法就是 Java 的 Integer.bitCount() 的实现方法。
