CS:APP | C Programming Lab

概述

C Programming Lab 的要求是实现一个支持 FIFO 和 LIFO 的队列。本实验的实验材料已经提供了队列数据结构的框架，需要读者来自行完成对数据结构的各种操作。完成本实验需要掌握的基本编程技能有：

  • 精准的 C 语言内存管理；
  • 对 C 指针和字符串的操作；
  • 对常用数据结构实现的优化；
  • 编写鲁棒性强的数据结构代码。

实验手册和实验材料可以在这里查阅。

数据结构

由 value 字段可知，这个队列的结点的值是一个指向字符数组的指针（也就是 C 语言对于字符串的表示），而 next 字段则是指向下一个结点的指针。这个队列的最上层表示是 queue_t，它只包含了一个类型为 list_ele_t 的指针，指向队列的头结点。队列的最后一个结点的 next 字段应当是 NULL。为了后续实现的进一步优化，读者可以自行在 queue_t 结构里添加新的字段，但不能改变 list_ele_t 的结构。

请注意区分 queue_t == NULL 和 q->head == NULL。前者表明该队列是一个无效队列，后者表明该队列是一个空队列。在后续的代码中读者需要辨别这两种不同的情况。

任务

读者的任务是修改 queue.h 和 queue.c，使得队列能正常工作。这里需要实现下面几个函数：

  • queue_new：创建一个新的空队列；
  • queue_free：删除队列，释放全部内存；
  • queue_insert_head：在队头插入一个元素（用于 LIFO）；
  • queue_insert_tail：在队尾插入一个元素（用于 FIFO）；
  • queue_remove_head：删除队头的元素；
  • queue_size：计算队列长度；
  • queue_reverse：反转队列（对结点指针操作，不可以重新分配内存）。

队尾插入一个元素的时间复杂度要求 $O(1)$；
计算队列长度的时间复杂度要求 $O(1)$；
反转队列的空间复杂度要求 $O(1)$，时间复杂度要求优于 $O(n^2)$。

插入元素（LIFO/FIFO）

在队列头插入元素操作流程是：

  1. 新建一个元素并分配内存空间；
  2. 为字符串分配内存空间，把 const char * 类型的参数复制到 char * 类型的 value 字段；
  3. 把新元素插入队列头，元素计数 + 1。

这里每一次分配内存空间都需要判断 malloc 返回的结果。为字符串分配空间时，若内存空间不足，则应当先释放队列元素的空间再返回 false。插入队列时，如果队列不为空，则直接把元素添加到队列头并更新 q->head 字段；如果队列为空，则让 q->head 和 q->tail 都指向新元素。

在队列尾插入元素操作流程与上一操作类似。由于增添了 q->tail 字段，该算法的时间复杂度为 $O(1)$。

删除队头的元素

删除队头元素（弹出元素）的操作流程是：

  1. 把队头元素的值复制到长度为 bufsize 的缓冲区 buf，buf 的最后一个字节应该是 \0；
  2. 删除队头元素，元素计数 - 1。

因为缓冲区的长度是有限的，所以这里不能直接用 strcpy，而应采用 strncpy 来复制给定长度的内存区域。

反转队列

用头插法反转队列。需要把队尾元素挨个插入到队头，当新的队头和旧的队尾相等时循环结束。反转队列的空间复杂度为 $O(1)$，时间复杂度为 $O(n)$。

这里反转队列（链表）使用了头插法。事实上，如果用递归的方法反转队列则会导致递归栈快速膨胀，空间复杂度为 $O(n)$，满足不了实验要求的原地操作。

思考

在进行复制内存的操作时，我一开始使用了 memcpy 函数，事实证明两者从结果上看并没有什么差别。尽管如此，我还是希望对 memcpy 和 strcpy 的效率进行比较。

memcpy 和 strcpy 源码比较

首先查阅 Linux 的 strcpy 实现，这个函数仅仅是按字节复制内存，到 \0 终止。

再查阅 Linux 的 memcpy 实现：

!(((unsigned long) dest ^ (unsigned long) src) & 7) 的含义是判断 dest - src 是否是 8 的整数倍。继续考察 __memcpy_aligned_up：

这段代码的含义就是以 8 字节为一个块批量复制内存。汇编代码 __asm__("ldq %0,%1":"=r" (tmp):"m" (*(unsigned long *) s)); 的操作是一次性复制四个字（Load Quadword），即 8 字节。

综合来看，strcpy 是按照字节复制的，到 \0 即停止；而 memcpy 是按照四个字批量复制的，在待复制内存长度较大的情况下对比 strcpy 会有明显的效率提升。当 n 比较大时，计算字符串长度的一次性开销足以忽略。
